# 讀書心得「質感」升級計畫

> 目的：讓 `/rui-xuan-book` 產出的讀書心得，不只「像你」，而是「想的也是你」。
> 狀態：計畫草案，待 ray 審核後再實作。
> 不在範圍：重讀 `chat/`（對這個目標沒用，省下好幾天）。

---

## 1. 問題與診斷

`/rui-xuan-book` 目前只吃一份**文體規格**（`rui-xuan-book.md`），它對兩件事是瞎的：

- 讀不到你那 14 篇既有心得 → 沒辦法**跨書串連**
- 讀不到你的真實人生 → 例子是「泛泛聰明讀者」的，不是你活過的

所以輸出能套對「形」（短行、句尾不留標點、關鍵句獨立），但「質」不對。

落差層級（ray 已確認落在第二格）：

| 落差在哪 | 該動的槓桿 | 本計畫處理？ |
|---|---|---|
| 形：節奏斷句 | 繼續調 `rui-xuan-book.md`（已在做） | 不動 |
| **質／連結：像泛泛讀者、沒織進其他書、沒你人生的味道** | 接上書庫＋生命素材＋補「質」規則 | ✅ 本計畫 |
| 想法本身：寫出你沒有的結論 | 改工作流，只上色不生成 | 暫不處理 |

「對的聲音」長怎樣，校準自 `快思慢想.md`（6/27 重寫版），它的質感指紋有三個：
1. **跨書串連**：一篇心得牽進《神經可塑性》《我可能錯了》＋Frankl「刺激與反應之間的空間」，每個連結都有動機
2. **挑人不挑實驗**：最有感的是康納曼自己也中招、還願意承認
3. **親歷當血肉**：「沉沒成本」落地成「那段早就該結束的關係」

## 4. 「動筆前先讀」wiring（可直接貼進 skill 開頭）

```markdown
## 動筆前先讀（這步決定心得「想的是不是你」）

在寫任何一個字之前：

1. 讀過 `book-reader/` 底下**所有**書本心得 `.md`（你讀過的書、你對每本書的真實角度）
   —— 這是跨書串連的素材庫，連結要從這裡長出來，不能憑空捏造書名
2. 讀 `book-reader/is-me/Rui-Xuan-生命素材.md`
   —— 這是你人生的血肉，抽象概念要落地時，從這裡伸手拿真實的事

讀完才開始寫。
```

> 用 glob「所有 `.md`」而不是寫死 14 個檔名，之後加書自動納入。

## 5. 新增的「質」規則（可直接貼進 skill）

```markdown
## 寫作實質（質）：比文體更重要的一層

**跨書串連要有動機，不是名詞亂灑**
- 只有當另一本書能「解釋這本書的機制」「回答這本書丟出的問題」或「把它的結論複雜化」時，才牽進來
- 牽進來要講清楚它補上了什麼（例：用《我可能錯了》補上「那半秒該做什麼」；用《神經可塑性》解釋「系統一為什麼那麼快」）
- 一篇最多牽 1–2 本，寧缺勿濫；連不出有動機的，就不要連

**挑人，不挑實驗**
- 讓你停下來的通常是人性、反諷、謙卑的時刻，不是技術細節或數據
- 找出這本書最「人」的那個轉折（例：康納曼自己也掉進促發效應、還願意公開承認），從那裡進去

**用親身經歷當血肉**
- 抽象概念要落地時，從生命素材拿你真實活過的事當例子
- 但用「你／我們」寫出來，讓它普世、可代入，而不是告白
  （例：「沉沒成本」→「那段早就該結束的關係」，不是「我跟某某分手」）
```

## 6. `Rui-Xuan-生命素材.md` 草稿（這頁長這樣）

> 這頁只給 skill 讀，不發佈。用法：寫到某個抽象概念、需要一個具體例子時，
> 從這裡挑一條真實的事，再用「你／我們」改寫成普世句。
> 私下記錄是真實的；輸出時抽象化，不指名道姓。

```markdown
# Rui-Xuan 生命素材（讀書心得的血肉來源）

| 主題（真實的事） | 可接到的概念 | 落地句範例（你／我們，句尾不留標點） |
|---|---|---|
| 那段早就該結束、卻放不下的關係 | 沉沒成本、損失趨避、執著、面子 | 那段早就該結束的關係，我們還是繼續往裡面丟自己 |
| 一個人去日本，去了五次以上 | 自我重整、獨處、找回主體性 | 有些路要一個人走，才聽得見自己原本的聲音 |
| 同事不思考、把責任推給你 | ownership、把腦袋外包、成熟 | 把腦袋外包出去的人，總有一天要別人替他收拾 |
| 養貓 ㄤ堆（膽小、社恐、無條件） | 穩定的情感、被需要、不求回報的陪伴 | 牠不懂你今天值不值得被愛，牠只是靠過來 |
| 僵直性脊椎炎、心悸、睡不好、疲勞 | 身體的有限、與身體和解、活在當下 | 身體會在你最不想聽的時候，提醒你它也有意見 |
| 演唱會票券司法、帳戶凍結、等不起訴 | 失控、制度、資源被卡住 | 有些失控不是你的錯，但收拾的還是你 |
| 想要空間、又極度想被穩定地愛 | 自由 vs 歸屬、安全地連結 | 我們說自己要的是自由，其實要的是能安全地靠近 |
| 把每次痛苦整理成文字、表格、分析 | 理智化、用秩序對抗混亂 | 把痛苦拆成條列，是因為整齊的東西比較不會痛 |
| 大量用 AI 重構工作、做內部訓練 | 工具、效率、思考密度 | 工具換了，問題沒換：你還是得自己決定要往哪想 |
```

> 草稿先列 10 條，涵蓋感情／家庭／工作／身體／制度／內在六條線。
> 審核時可增刪、可改落地句的口吻。

## 7. 驗證方式（建議排進實作）

挑一本**你讀過、但還沒寫成心得**的書，做 A/B：

1. 用「現在的 skill」生一篇
2. 用「接好書庫＋生命素材＋質規則」的 skill 生一篇
3. 並排看：第二篇有沒有出現有動機的跨書連結、有沒有你人生的味道

通過標準：第二篇讀起來「想的是你」，而不只是「像你」。

## 8. 風險與「不做的事」

- **跨書串連要克制**：1–2 個、有動機才連，否則比沒連更糟（變名詞亂灑）

- **形那層不要動**：你 6/27 調好的短行／無句尾標點繼續沿用
